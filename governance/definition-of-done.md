# Definition of Done

A UI task is done when:

- the requested problem is solved
- repository fit is preserved
- the most relevant source-of-truth files were respected
- states were reviewed and handled
- responsive behavior was considered
- design-system alignment was maintained or improved
- accessibility regressions were not introduced
- the change is small, clear, and reviewable

## Review prompts

- Would another contributor understand why this shape was chosen?
- Does the solution reuse what already exists?
- Did the task quietly create design or architecture debt?
- Are state, accessibility, and responsive behavior treated as part of correctness?
