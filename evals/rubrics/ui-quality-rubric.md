# UI Quality Rubric

## Score 1 — Poor

The output solves the request superficially but introduces hierarchy problems, incomplete states, or clear repository mismatch.

## Score 3 — Acceptable

The output is usable and mostly aligned but has avoidable rough edges in structure, states, or system consistency.

## Score 5 — Strong

The output is clear, coherent, repository-native, and improves usability without creating design or engineering debt.
