# Workflow — Propose Change

## Goal

Change canonical guidance without creating overlap, duplication, or unstable policy.

## Sequence

1. identify the recurring problem
2. find the current canonical doc that is closest to the issue
3. verify that the problem is not better solved in a workflow, skill, agent, hook, or script
4. update one canonical document or one tightly related set of documents
5. update cross-references
6. update scripts or hooks when enforcement needs to change
7. record the rationale in the pull request or governance note
