# Form Patterns

Forms should reduce ambiguity and make successful completion feel straightforward.

## Structure

A good form usually includes:

1. purpose or heading
2. grouped fields
3. helper text where needed
4. clear validation
5. primary submit action
6. secondary cancel or back action when relevant

## Rules

- group related fields together
- label every input clearly
- avoid placeholder-only labeling
- validation should appear close to the field
- preserve user input during validation and error recovery
- large forms should use sectional hierarchy

## Submission states

Forms should define:

- idle
- field validation
- submitting
- success or confirmation
- error and recovery

## Review step

Use review or confirmation steps only when the task has real consequence or high complexity.
