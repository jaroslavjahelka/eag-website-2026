# UI Audit

## Use this skill when

- the user asks for review rather than full implementation
- a page feels inconsistent or hard to use
- a refactor needs evidence before code changes

## Sequence

1. Identify the target page, component, or flow.
2. Review against the checklist reference.
3. Group findings by severity: blocker, important, improvement.
4. Separate structural issues from styling symptoms.
5. Recommend the smallest set of changes that meaningfully improves quality.

## Output requirements

- explain the underlying issue, not just the symptom
- tie recommendations back to system rules
- avoid generic advice with no repository fit
