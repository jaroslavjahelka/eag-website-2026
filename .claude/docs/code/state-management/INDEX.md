# State management

An interaction component or page is incomplete if its meaningful states are undefined.

---

## References

- [state-core.md](./state-core.md) – Components and routes core states
- [state-loading.md](./state-loading.md) – Loading state knowledge
- [state-status.md](./state-status.md) – Status, error handling knowledge

---

## Rules

- every interactive component should has states
- every page should define at least loading, empty, success or confirmation, and error where relevant
- states should be visible both visually and behaviorally
